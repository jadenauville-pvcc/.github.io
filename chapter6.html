Chapter 6: Programming Languages

    Machine Language

    • Sequences of instructions that are encoded in numeric digits.

    Debugging

    • Locating and correcting errors in machine languages within a program.

    Assembler

    • Developed to convert mnemonic expressions into machine language instructions.

    Assembly Language

    • A mnemonic system for representing programs.

    Translator

    • Translates programs expressed in high-level primitives into machine-language programs.

    Compiler

    • Compiles several machine instructions into short sequences to simulate the activity requested by a single high-level primitive.

    Interpreter

    • An alternative to translators, emerged as another means of implementing third-generation languages.

    Cross-platform Software

    • Software that is independent of an operating system's design as well as the machine's hardware design and is therefore executabl throughout a network.

    Natural Language

    • Languages such as English, Spanish, and French.

    Formal Language

    • Languages such as programming languages, like Java, C, and Python.

    Machine Independence

    • Does not rely on characteristics of a particular machine.

    High-Level Programming Language

    • Provides more abstractions for the programmer and make it easier for people to read and write a program.

    Object-Oriented Programming (OOP)

    • Software system is viewed as collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects.

    Scripted Language

    • Languages used to perform administrative tasks rather than to develop complex programs.

    Data Types

    • Integer: Refers to numeric data consisting of whole numbers.
    • Float: Refers to numeric data that might contain values other than whole numbers, probably stored in floating-point notation.
    • String: A data type used in programming, that is used to represent text rather than numbers.

    Primitive Data Type

    • Data types that are included as primitives in a programming language.

    Data Structure

    • The conceptual shape or arrangement of data.

    Python Collection Data Structures

    • List: Lists are used to store multiple items in a single variable. Created using square brackets.
    • Tuple: Used to store multiple items in a single variable, ordered and unchangeable. Written with round brackets.
    • Set: Used to store multiple items in a single variable. It is unordered, unchangeable, and unindexed. Written with curly brackets.
    • Dictionary: Used to store data values in key:value pairs. It is ordered, changeable and do not allow duplicates. Written with curly brackets, and have keys and values.

    Function

    • A set of instructions for performing a task that can be used as an abstract tool by other program units.

    Local Variable

    • A variable declared within a function that can only be referenced within that function.

    Global Variable

    • Variables whose scopes are not restricted to a particular part of a program.

    Source Program

    • The program in its original form.

    Object Program

    • The program in its translated version.
